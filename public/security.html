RobustOTP Security Architecture

Enterprise-level abuse resistant authentication parameters

Cryptographic Hashing Bcrypt

Plaintext OTP codes are never written to disk or logged. They are stored as Bcrypt hashes (using a cost parameter of 12) immediately upon generation. Comparison is performed in constant time to defeat side-channel timing analysis.

CSPRNG Generation Secure Random

Verification keys are generated using a cryptographically secure pseudorandom number generator (CSPRNG via Node's crypto.randomInt). This ensures complete unpredictability and guards against sequence scanning.

Brute Force Lockout Auto Lock

To secure the numeric space, OTP sessions automatically lock after 5 failed verification attempts. Expired or used sessions are immediately invalidated to prevent reuse or replay attacks.

Dynamic Rate Limiting IP Guard

Includes built-in IP-based rate limiting on all login routes, combined with client limits of 3 requests per 5 minutes to prevent email flooding abuse and resource exhaustion.

Session Isolation State Locked

Every OTP challenge is isolated and state-serialized. Active verification tokens are bound to a single unique session token, preventing cross-user session tampering and token hijacking.

Auto-Purge Hygiene Storage Clean

Expired authentication hashes are automatically pruned from the database during keep-alive cycles, maintaining a clean and secure database storage footprint.

Decoupled Authentication Architecture

RobustOTP operates as an isolated authentication authority. Offloading OTP logic to a dedicated microservice improves security:

  • Isolated Threat Surface: Authentication credentials and hash comparisons run inside a locked-down microservice, separate from your core application logic.
  • Zero Local DB Bloat: Eliminates the need to pollute your primary user tables with short-lived codes, lockout tallies, and expiry timestamps.
  • State Serialization: Session validation tokens are serialized and verified independently of your main user profiles, preventing cross-session privilege escalation.
